# AnalyzeGit - Technical Architecture & Workflow

AnalyzeGit is a high-performance auditing system designed to evaluate GitHub repositories and developer personas using AI-driven heuristic analysis.

## 🏗️ System Architecture

The project follows a distributed, event-driven architecture designed for scalability and real-time user feedback.

### 1. Request Handling & Event Streaming
- **Frontend (Client)**: A React-based SPA that communicates with the API via REST for authentication and history, and via **WebSockets (Socket.io)** for analysis streaming.
- **Backend (API Gateway)**: An Express.js server that validates incoming GitHub URLs, manages JWT-based authentication, and acts as a producer for the background processing queue.

### 2. Background Processing (Producer-Consumer)
- **Task Queue (BullMQ/Redis)**: To ensure the API remains responsive, all compute-intensive analysis tasks are offloaded to a Redis-backed priority queue.
- **Worker Service**: A dedicated consumer that monitors the queue, processes jobs asynchronously, and updates the task status. Integration with **Upstash Redis** allows for reliable cross-process communication even in cloud environments.

### 3. AI Analysis Engine
- **Data Acquisition**: The worker uses **Octokit** (GitHub REST API) to fetch repository metadata, commit history, file structures, and user activity profiles.
- **Heuristic Auditing (Gemini 2.5 Flash)**: The raw GitHub data is structured and passed to the Gemini LLM with custom-engineered prompts. The model performs:
  - Architectural pattern identification.
  - Code quality scoring.
  - Professional persona evaluation.
- **Persistence**: Final audit reports are stored in **MongoDB** for historical retrieval and PDF export generation.

## 🔄 Technical Workflow

1. **Submission**: User submits a GitHub URL. The API creates a unique `Job ID` and pushes a packet to the Redis queue.
2. **Buffering**: The worker picks up the job. The status is updated to `analyzing`.
3. **Execution**:
   - Worker fetches GitHub metadata.
   - Data is summarized and sent to the Gemini API.
   - Gemini returns a structured JSON audit (Score, Summary, Strengths, Improvements).
4. **Real-time Update**: As the worker completes phases, it emits events back to the Client via the shared **Socket.io** namespace.
5. **Completion**: The frontend receives the `analysisComplete` event, triggers the auto-scroll logic, and renders the persisted result.

## 🛠️ Data Model
- **History Collection**: Stores `userId`, `url`, `type` (User/Repo), `metadata` (Avatar, Name, Bio), and the `analysis` object (JSON generated by AI).

## 🔌 API & Integration Points
- **Gemini 2.5 Flash**: Core reasoning engine.
- **Socket.io**: Real-time state synchronization.
- **jsPDF-autotable**: Dynamic PDF generation from structured JSON.
